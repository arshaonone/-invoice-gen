fix: PDF generation "Failed to generate PDF" error

Root cause: downloadPDF was attempting to show the inner printRef div,
but the outer container had display:none (from Tailwind's hidden class),
which blocked html2canvas from rendering anything.

Fix:
- Added printContainerRef to the outer wrapper div
- Changed outer div from className="hidden" to style={{ display: 'none' }}
  so JavaScript can reliably toggle it
- downloadPDF now shows/hides the OUTER container (not the inner div)
- Container is revealed off-screen at 794px width for A4 capture
- Container is guaranteed to be re-hidden even on error (finally block)
- Added allowTaint: true and windowWidth: 794 to html2canvas options
